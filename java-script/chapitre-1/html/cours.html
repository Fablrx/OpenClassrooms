Le type de chaîne

Cette valeur représente un texte. Elle est déléimité par une paire de guillemet doubles : "Ceci est une chaine"

On peut utiliser les \ pour utiliser des caractères spéciaux comme par exemple \n pour faire un retour à la ligne dans une chaîne.

Voir l'exercice 2 dans le fichier JavaScript avec le code Ceci est une chaine \n Sur plusieurs lignes.

On ne peut pas additionner ou soustraire des valeurs de type chaîne comme on peut le faire des nombres. En revanche, l'opérateur + peut être appliqué à deux valeurs de type chaîne. Son résultat est la jointure de ces deux chaînes, appelée concaténation.

La notion de variable

Rôle des variables

Un programme informatique mémorise des données en utilisant des variables. Une variable est une zone de stockage d'information. On peut l'imaginer comme une boîte dans laquelle on range des choses.

Propriétés d'une variable

Une variable possède trois grandes propriétés.

Propriétés

Définition

NOM

Permet de l'identifier. Un nom de variable peut contenir des lettres majuscules ou minuscules, des chiffres (sauf en première position) et certains caractères comme le dollar ($) ou le tiret bas, appelé underscore (_).

VALEUR

C'est la donnée actuellement mémorisée dans la variable.

TYPE

Détermine le rôle et les opérations applicables à cette variable.

Avant de pouvoir stocker des informations dans une variable, il faut la créer. Cette opération s'appelle la déclaration de la variable.

La notion d'expression

Une expression est un morceau de code qui produit une valeur.
On crée une expression en combinant des variables, des valeurs et des opérateurs. Toute expression produit une valeur et correspond à un certain type. Le calcul de la valeur d'une expression s'appelle l'évaluation. Lors de l'évaluation d'une expression, les variables sont remplacées par leur valeur.

3 est une expression dont la valeur est le nombre :

var c = 3;

d est une expression dont la valeur est celle de c (ici 3) :

var d = c;

(d + 1) est une expression. Sa valeur est celle de d augmentée de 1 (ici 4)

d = d + 1;
console.log(d);

Une expression peut comporter des parenthèses qui modifient la priorité des opérations lors de l'évaluation. En l'absence de parenthèses, la priorité des opérateurs est la même qu'en mathématiques.

Contient la valeur 11 :

var e = 3 + 2 * 4;

e contient la valeur 20

e = (3 + 2) * 4;

Conversion de types

L'évaluation d'une expression peut entraîner des conversions de type. Ces conversions sont dites implicites : elles sont faites automatiquement, sans intervention du programmeur.

var f = 100; console.log("La variable f contient la valeur" + f);

Le rendu :

Le langage JavaScript est extrêmement tolérant au niveau des conversions de type. Cependant, il arrive qu'aucune conversion ne soit possible. En cas d'échec de la conversion d'un nombre, la valeur du résultat est NaN (Not a Number).

Par exemple :

var g = "cinq" * 2;
console.log(g);

Pour forcer la conversion d'une valeur dans un autre type (nombre ou chaîne de caratère), On peut utiliser des conversions explicites comme Number() et String().

Par exemple :

var h = "5";
console.log(h + 1);

Cette concaténation affiche 51

h = Number("5");
console.log(h + 1)

Cette addition numériqe affiche 6

Saisie d'un nombre

prompt() renvoie toujours une valeur de type chaîne. c'est pour celà qu'on utilise number() pour convertir la valeur en nombre pour la comparer à d'autres nombres ou l'utiliser dans les expressions mathématiques.

Voici un exemple de code :

var saisie = prompt("Entrer un nombre : ");

Cette saisie est de type chaîne.

var nb = number(saisie);

nb est de type nombre

Il est possible de combiner les deux opérations (saisie et conversion) en une seule ligne de code, pour un résultat identique :

var nb = Number(prompt("Entrez un nombre : "));

Ici, le résultat de la saisie utilisateur est directement converti en une valeur de type nombre par l'ordre Number() et affecté à la variable nb.

Importance du nommage des variables

La question est la suivante : Comment faire pour bien nommer les variables de ses programmes ?

Choisir des nomes significatifs

Donner à une variable un nom qui reflète son rôle !
Exemples :

  • rayon
  • pi
  • perimetre

Bannir les caractères accentués

Ne pas utiliser les noms réservés du langage

Vous trouverez sur cette page la liste des noms réservés de JavaScript.

Adopter une convention de nommage

Nous pouvons utiliser la norme camelCase (appelée parfois lowerCamelCase). Elle repose sur deux grands principes :

  • Tout nom de variable commence par une lettre minucscule.
  • Si le nom d'une variable se compose de plusieurs mots, la première lettre de chaque mot (sauf le premier) s'écrit en majuscule.

Par exemple, les noms montantTravauxMaison ou codeClientSuivant respectent la norme camelCase.

La notion de condition

Une condition est une expression dont l'évaluation produit une valeur soit vraie, soit fausse : on parle de valeur booléenne.

Quand une valeur d'une condition est vraie, on dit qu'elle est vérifiée.

nombre et chaîne font partie du même type que booléen. Ce type n'a que deux valeurs possibles : true et false.

if (true) {
}

la condition du if est toujours vraie, les instructions de ce bloc seront toujours exécutées

if (false){
}

la condition de if est toujours fausse, les instructions de ce bloc ne seront jamais exécutées.
